# Gitea Progress Report

## 功能描述
对每个 Gitea 可见仓库分别生成一份进度报告，内容包括：
- AI 综合评估（整体一段话）
- 成员贡献排行（带工作量占比）
- 每位成员的工作摘要、具体提交记录、改动文件
- 风险提示（低活跃成员、模糊提交等）

每个仓库单独发送一封邮件给该仓库的管理员。

## 使用场景
- 当用户说"帮我生成进度报告"、"发送周报"、"发送日报"时触发
- 定时任务：每天或每周自动生成并发送报告
- 当用户想了解团队最近工作情况时

## 使用方法

```bash
# 对所有可见仓库生成报告（过去7天）
python scripts/generate_report.py --hours 168

# 对所有可见仓库生成报告（过去24小时）
python scripts/generate_report.py --hours 24

# 对指定仓库生成报告
python scripts/generate_report.py --hours 168 --repo mayidan/project-test
```

## 执行流程

第一步：运行 Python 脚本获取结构化数据摘要。

第二步：收到数据后，对每个仓库分别执行以下操作：

### 1. 生成报告

用以下格式生成纯文字报告（严格按照格式，不要用 Markdown）：

```
╔══════════════════════════════════════════════════════════╗
  项目进度报告  |  {仓库名}  |  {generated_at 日期部分}
╚══════════════════════════════════════════════════════════╝

【本期总览】
提交次数：{total_commits} 次   参与成员：{total_members} 人
新增代码：+{total_additions} 行   删除代码：-{total_deletions} 行
统计周期：{time_range}

【AI 综合评估】
{根据提交内容和成员活跃度，用2-4句话综合评估本期项目进展。
重点说明：整体进展如何、主要推进了哪些工作、有无值得关注的问题。
语言简洁专业，避免废话。}

══════════════════════════════════════════════════════════

【成员贡献排行】
{按提交次数从高到低排列，每行格式如下：}
  #{排名}  {成员名}  {提交次数}次提交  +{additions}行/-{deletions}行  {占总提交比例的进度条} {百分比}%

进度条规则：总长度10格，用█表示占比，例如67%显示为██████████

══════════════════════════════════════════════════════════

【成员详情】
{对每位成员按以下格式输出：}

▌{成员名}                              提交 {X} 次 | {进度判断}
  ┌─────────────────────────────────────────────────────┐
  │ 工作摘要：{根据该成员所有commit message提炼的一句话总结} │
  └─────────────────────────────────────────────────────┘
  提交记录：
  · [{时间 MM-DD HH:mm}]  {commit message}
                   改动文件：{filename}(+{additions}/-{deletions}) ...

{每位成员之间用横线分隔：}
──────────────────────────────────────────────────────────

进度判断规则：
- 提交次数 >= 3 且无模糊提交：✅ 持续推进
- 提交次数 >= 3 但有模糊提交：✅ 持续推进（建议规范提交信息）
- 提交次数 1-2 次：⚠️ 活跃度偏低
- 提交次数 0 次：❌ 本期无提交

══════════════════════════════════════════════════════════

【风险提示】
{如果有以下情况则列出，每条前加对应图标：}
  ⚠️  {成员名} 本期仅提交 {X} 次，建议关注进度
  ⚠️  发现 {X} 条模糊提交（如"fix"、"update"），建议规范提交信息
  ℹ️  {其他值得注意的情况}
{如果没有任何风险，输出：本期暂无风险提示。}

══════════════════════════════════════════════════════════
  由 AIFusionBot 自动生成 · {generated_at}
  查看详情：{GITEA_URL}/{仓库名}
══════════════════════════════════════════════════════════
```

### 2. 发送邮件

生成报告后立即调用 imap-smtp-email skill 发送邮件：
- 收件人：数据中的 admin_email 字段
- 邮件主题：【项目进度报告】{仓库名} · {time_range}
- 邮件正文：上面生成的完整报告文本

## 参数说明
- `--hours`：时间范围，24=过去24小时，168=过去7天，默认168
- `--repo`：指定仓库，格式 owner/reponame，不填则处理所有可见仓库

## 依赖 skill
- imap-smtp-email：发送邮件，请确保该 skill 已安装并配置好 SMTP 信息
