# Gitea Progress Report

## 功能描述
对每个 Gitea 可见仓库分别生成一份进度报告，内容包括：
- 按成员汇总的提交摘要和工作量统计
- 对每位成员工作进度的判断
- 风险提示（低活跃成员、模糊提交信息等）

每个仓库单独发送一封邮件给该仓库的管理员。

## 使用场景
- 当用户说"帮我生成进度报告"、"发送周报"、"发送日报"时触发
- 定时任务：每天或每周自动生成并发送报告
- 当用户想了解团队最近工作情况时

## 使用方法
```bash
# 对所有可见仓库生成报告（过去7天）
python scripts/generate_report.py --hours 168

# 对所有可见仓库生成报告（过去24小时）
python scripts/generate_report.py --hours 24

# 对指定仓库生成报告
python scripts/generate_report.py --hours 168 --repo mayidan/project-test
```

## 执行流程

第一步：运行 Python 脚本获取结构化数据摘要。

第二步：你收到数据后，对每个仓库分别执行以下操作：

1. 根据数据摘要，用中文生成一份纯文字进度报告，格式如下：
```
【项目进度报告】{仓库名} · {时间范围}
============================================

一、总览
- 总提交次数：X 次
- 参与成员：X 人
- 累计新增：+X 行 / 删除：-X 行

============================================
二、按成员汇总
--------------------------------------------
▶ {成员名}
  提交次数：X 次 | 新增：+X 行 | 删除：-X 行
  活跃分支：{分支名}
  主要工作：
  - {根据 commit message 总结的工作内容}
  进度判断：✅持续推进 / ⚠️活跃度偏低 / ❌本期无提交

（每位成员重复以上格式）

============================================
三、风险提示
- {低活跃成员提示}
- {模糊提交信息提示}

============================================
报告由 AIFusionBot 自动生成 | {generated_at}
```

2. 生成报告后，立即调用 imap-smtp-email skill 发送邮件：
   - 收件人：数据中的 admin_email 字段
   - 邮件主题：【项目进度报告】{仓库名}
   - 邮件正文：上面生成的报告文本

## 参数说明
- `--hours`：时间范围，24=过去24小时，168=过去7天，默认168
- `--repo`：指定仓库，格式 owner/reponame，不填则处理所有可见仓库

## 依赖 skill
- imap-smtp-email：发送邮件，请确保该 skill 已安装并配置好 SMTP 信息
